p1. 메인 로그인 페이지
  - 모든 사용자가 처음 사이트에 접속했을 때 만나는 페이지
○ 필요 요소 
  1) 로그인용 input 박스 2개
    ① 로그인용 휴대전화번호,
    ② 비밀번호
  2) 로그인 버튼 1개
    - 클릭 시 로그인 처리
    - 로그인한 계정이 장애인 이용자면, p2(장애인 이용자 홈)으로 이동
    - 로그인한 계정이 서포터면, p9(서포터 홈)으로 이동




p2. 장애인 이용자 홈
  - 장애인 이용자가 서포터 매칭을 요청하는 페이지
  - 카카오T 메인화면 참고
○ 필요 요소
  1) 잔여 포인트 출력
    - 화면 상단에 "현재 50000포인트 남음"이라는 텍스트 출력
  2) 출발/도착 입력을 위한 버튼 2개
    ① 출발지 입력 버튼: 클릭 시 p3(출발지 입력 페이지)로 이동, 출발지 입력이 완료되면 버튼의 이름을 입력된 출발지로 변경
    ② 도착지 입력 버튼: 클릭 시 p4(도착지 입력 페이지)로 이동, 도착지 입력이 완료되면 버튼의 이름을 입력된 도착지로 변경
  3) 편도/왕복 확인용 라디오버튼 2개
    ① 편도 라디오버튼: default
    ② 왕복 라디오버튼
  4) 이동수단 선택 라디오버튼
    ① 도보 라디오버튼: default
    ② 대중교통(교통비 이용자 부담) 라디오버튼
    ③ 택시(교통비 이용자 부담) 라디오버튼
  5) 즉시 매칭 안내 텍스트
    - "* 기본적으로 매칭은 즉시 이루어지며, 시간을 지정하고 싶으신 경우 아래의 시간지정 버튼을 클릭해 시간을 선택해주세요."라는 텍스트 출력
  6) 시간지정 버튼
    - 클릭 시 p5(시간지정 선택 페이지)로 이동, 시간 입력이 완료되면 버튼의 이름을 입력된 시간으로 변경
  7) 지원 내용 선택박스(selection)
    - default 옵션은 "단순이동지원"
    - 옵션은 "단순이동", "가사보조", "외출지원"
  8) 상세 요청사항 텍스트박스(textarea)
    - placeholder 속성으로 "필요한 경우 상세한 요청사항을 입력해주세요."라는 값 지정
  9) 매칭 시작 버튼
    - 하단에 위치하며 클릭 시 p6(장애인 이용자 매칭 시도 페이지)로 이동




p3. 출발지 입력 페이지
○ 필요 요소
  1) 출발지 입력용 input박스
    - 지도나 택시 앱의 출발지 검색창과 비슷한 느낌
  2) 검색 버튼(type="submit")
    - 클릭 시 하단에 검색 결과가 출력됨
  3) 검색 결과 영역
    - 임시로 "대구대학교ㅛ 정문"에 대한 검색결과 출력
    - 출력 예시 데이터(3개): "대구대학교 정문2, 경북 경산시 진량읍", "CU 대구대정문점, 경북 경산시 진량읍",, "GS25 대구대정문점, 경북 경산시 진량읍"
    - 출력되는 요소는 모두 클릭이 가능한 링크 또는 버튼으로 만들고, 클릭 시 p2(장애인 이용자 홈)으로 이동




p4. 도착지 입력 페이지
○ 필요 요소
  * p3(출발지 입력 페이지)와 동일




p5. 시간지정 선택 페이지
○ 필요 요소
  - 카카오T의 벤티 예약 화면 참고
  1) 시간지정 날짜/시/분 선택 영역
    ① 날짜 selection: 옵션은 2개 -> "오늘", "내일"
    ② 시 슬라이더(input type="range"): min=0, max=23
    ③ 분 슬라이더(input type="range"): min=0, max=59
  2) 선택완료 버튼
    - 클릭 시 p2(장애인 이용자 홈)으로 이동
    - 입력된 시간이 현재 시각보다 30분 이후여야 함. 그렇지 않으면 이 버튼은 비활성화 처리(클릭불가)




p6. 장애인 이용자 매칭 시도 페이지
○ 필요 요소 
  1) 타이틀 텍스트
    - 화면 상단에 큰 글씨로 "서포터 매칭 중" 출력
  2) 요청사항 출력 영역
    ① 출발지
    ② 도착지
    ③ 편도 or 왕복
    ④ 이동수단
    ⑤ 즉시 or 시간지정(ex: "오늘 / 13시 20분")
    ⑥ 지원 내용
    ⑦ 상세 요청사항(없으면 빈 문자열)
  3) 요청 중단 버튼
    - 클릭 시 p2(장애인 이용자 홈)으로 이동




p7. 장애인 이용자 매칭 성공 페이지
○ 필요 요소 
  1) 타이틀 텍스트
    - 화면 상단에 큰 글씨로 "서포터 매칭 완료!" 출력
  2) 연결된 서포터 정보 영역
    ① 이름
    ② 휴대전화번호
  3) 요청사항 출력 영역
    ① 출발지
    ② 도착지
    ③ 편도 or 왕복
    ④ 이동수단
    ⑤ 즉시 or 시간지정(ex: "오늘 / 13시 20분")
    ⑥ 지원 내용
    ⑦ 상세 요청사항(없으면 빈 문자열)
  4) 매칭 취소 버튼
    - 클릭 시 p8(장애인 이용자 매칭 취소 확인 페이지)으로 이동




p8. 장애인 이용자 매칭 취소 확인 페이지
○ 필요 요소 
  1) 텍스트 출력 "연결된 매칭을 취소하시겠습니까? 패널티가 발생할 수 있습니다."
  2) 네/아니요 버튼
    - "네" 클릭 시 p2(장애인 이용자 홈)으로 이동, "아니요" 클릭 시 p7(장애인 이용자 매칭 성공 페이지)로 이동




p9. 서포터 홈
  - 서포터가 매칭 기록을 조회하고 매칭 여부를 설정하는 페이지
○ 필요 요소
  1) 획득 포인트 출력
    - 화면 상단에 "50000포인트 획득"이라는 텍스트 출력
  2) 텍스트 출력 "지금까지 총 3명의 장애인 이용자를 서포트했습니다."
    - 숫자 3의 경우 state를 이용해 출력
  3) 최근 매칭 기록 5개 출력 영역
    - 한 줄에 번호, 매칭시각, 소요시간, 출발지, 도착지, 지원 내용 출력
    - 총 5줄 출력
  4) 매칭 여부 전환 버튼
    - "여기를 클릭하여 매칭 상태를 ON으로 변경하세요."라는 텍스트를 출력하는 버튼
    - 하단에 위치하며 클릭 시 "현재 매칭 상태가 ON입니다. 여기를 클릭하면 OFF 상태로 전환됩니다."로 텍스트가 변경됨




p10. 서포터 매칭 성공 페이지
○ 필요 요소 
  1) 타이틀 텍스트
    - 화면 상단에 큰 글씨로 "이용자 매칭 완료!" 출력
  2) 연결된 이용자 정보 영역
    ① 성별
    ② 휴대전화번호
  3) 요청사항 출력 영역
    ① 출발지
    ② 도착지
    ③ 편도 or 왕복
    ④ 이동수단
    ⑤ 즉시 or 시간지정(ex: "오늘 / 13시 20분")
    ⑥ 지원 내용
    ⑦ 상세 요청사항(없으면 빈 문자열)
  4) 매칭 취소 버튼
    - 클릭 시 p11(서포터 매칭 취소 확인 페이지)으로 이동




p11. 서포터 매칭 취소 확인 페이지
○ 필요 요소 
  1) 텍스트 출력 "연결된 매칭을 취소하시겠습니까? 패널티가 발생할 수 있습니다."
  2) 네/아니요 버튼
    - "네" 클릭 시 p9(서포터 홈)으로 이동, "아니요" 클릭 시 p10(서포터 매칭 성공 페이지)로 이동
